Subscription

Subscription

new Subscription(subscriptionId, opts)

The Subscription class manages the state and events of a subscription.

You can use the Subscription class to define application functionality when specific events occur, like receiving a message, or when the subscription enters different subscription state machine state.

When the application receives a message as subscription data, the data event occurs and the received message is passed, as a Protocol Data Unit (PDU), to the fn parameter of the Subscription.on(event, fn) event handler method.

Use the Subscription.on(event, fn) method to add an event handler to the subscription, which 'fires' when the subscription receives a published message. The event parameter in this method can take a value of data or the value of the action element of the PDU, for example, rtm/subscription/data.

You can also set event handlers when the subscription enters or leaves a state in the subscription state machine. A subscription can be in one of the following states: subscribing, subscribed, unsubscribing, unsubscribed, or failed.

See State Machines in the online docs.

Note: When the connection from the RTM client to RTM drops, all subscriptions are unsubscribed, and then resubscribed when the connection is restored. You can direct the JavaScript SDK to automatically reconnect or you can detect the disconnect manually and then reconnect to RTM at the proper message stream position with the position parameter.

Example
// create a new subscription to the 'your-channel' channel
var subscription = rtm.subscribe('your-channel');


subscription.on('rtm/subscription/data', function (pdu) {
    pdu.body.messages.forEach(console.log);
});
subscription.on('enter-subscribed', function () {
    rtm.publish('your-channel', {type: 'init'});
});
subscription.on('data', function (pdu) {
    if (pdu.action.endWith('/error')) {
        rtm.restart();
    }
});

fire(name) → {void}

Executes all handlers attached to the given event type.

The event name is one of the client or subscription state machine events, for example, data or enter-connected for the RTM client or enter-subscribed for a Subscription object.
